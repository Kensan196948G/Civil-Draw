## ⚠️ リスクと対応

| リスク | 影響 | 対応 |
|--------|------|------|
| Entra ID 統合の複雑性 | リリース遅延 | M2 前半で PoC / 社内 IT 部門と早期連携 |
| 大量図形パフォーマンス | 実用性欠如 | M2 中に早期ベンチマーク / 最悪時は図形上限緩和で妥協 |
| 現場 UX 不満 | UAT 不合格 | M4 α版で早期検知 / M3 で磨き込み |
| DXF 互換性 | AutoCAD 取込失敗 | 既存図面 10 件でのラウンドトリップ試験を M2 に組込 |
| 拠点環境依存 | 展開失敗 | M5 β で 3 拠点 (本社+2遠隔) で事前検証 |

## 🔄 Cron 実行・セッション運用

- **スケジュール**: Mon–Sat / 5 時間 / プロジェクト別
- **推定セッション数**: 月 24 × 6 = **約 144 セッション**
- **各セッションの運用**: Monitor → Development → Verify → Improvement をループ (回数は CTO 判断)
- **停止条件**: STABLE 達成 / Token 95% / 5 時間到達 / セキュリティ検知 / 同一原因 3 回
- **フェーズ変更の原則**: 進捗に応じて CTO 判断で可。ただし **2026-10-23 リリースは絶対厳守**
